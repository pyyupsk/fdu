Format dates and times using customizable token patterns. The `format()` method supports locale-aware formatting and escaped text for flexible output.

## Basic Usage

```typescript
import { fdu } from "@pyyupsk/fdu";

const date = fdu("2025-09-30T14:30:45");

date.format("YYYY-MM-DD"); // '2025-09-30'
date.format("MMMM DD, YYYY"); // 'September 30, 2025'
date.format("DD/MM/YYYY"); // '30/09/2025'
```

## All Format Tokens

### Year

| Token  | Output | Description  |
| ------ | ------ | ------------ |
| `YYYY` | 2025   | 4-digit year |
| `YY`   | 25     | 2-digit year |

### Month

| Token  | Output    | Description        |
| ------ | --------- | ------------------ |
| `MMMM` | September | Full month name    |
| `MMM`  | Sep       | Short month name   |
| `MM`   | 09        | 2-digit month      |
| `M`    | 9         | Month without zero |

### Day of Month

| Token | Output | Description      |
| ----- | ------ | ---------------- |
| `DD`  | 30     | 2-digit day      |
| `D`   | 30     | Day without zero |

### Day of Week

| Token  | Output | Description        |
| ------ | ------ | ------------------ |
| `dddd` | Monday | Full weekday name  |
| `ddd`  | Mon    | Short weekday name |
| `dd`   | Mo     | Min weekday name   |
| `d`    | 1      | Day of week (0-6)  |

### Hour

| Token | Output | Description             |
| ----- | ------ | ----------------------- |
| `HH`  | 14     | 2-digit hour (24h)      |
| `H`   | 14     | Hour (24h) without zero |
| `hh`  | 02     | 2-digit hour (12h)      |
| `h`   | 2      | Hour (12h) without zero |

### Minute

| Token | Output | Description         |
| ----- | ------ | ------------------- |
| `mm`  | 30     | 2-digit minute      |
| `m`   | 30     | Minute without zero |

### Second

| Token | Output | Description         |
| ----- | ------ | ------------------- |
| `ss`  | 45     | 2-digit second      |
| `s`   | 45     | Second without zero |

### Millisecond

| Token | Output | Description         |
| ----- | ------ | ------------------- |
| `SSS` | 123    | 3-digit millisecond |

### AM/PM

| Token | Output | Description |
| ----- | ------ | ----------- |
| `A`   | PM     | Uppercase   |
| `a`   | pm     | Lowercase   |

### Ordinal

| Token | Output | Description                  |
| ----- | ------ | ---------------------------- |
| `Do`  | 30th   | Day with ordinal (1st, 2ndâ€¦) |

## Escaped Text

Use square brackets to escape text:

```typescript
date.format("[Today is] YYYY-MM-DD"); // 'Today is 2025-09-30'
date.format("YYYY [year]"); // '2025 year'
date.format("[The time is] HH:mm"); // 'The time is 14:30'
```

## Common Patterns

```typescript
const date = fdu("2025-09-30T14:30:45");

// ISO format
date.format("YYYY-MM-DDTHH:mm:ss"); // '2025-09-30T14:30:45'

// US format
date.format("MM/DD/YYYY"); // '09/30/2025'

// European format
date.format("DD.MM.YYYY"); // '30.09.2025'

// Long format
date.format("MMMM DD, YYYY [at] hh:mm A"); // 'September 30, 2025 at 02:30 PM'

// Full format
date.format("dddd, MMMM DD, YYYY"); // 'Monday, September 30, 2025'

// Time only
date.format("HH:mm:ss"); // '14:30:45'
date.format("hh:mm A"); // '02:30 PM'

// With ordinal
date.format("MMMM Do, YYYY"); // 'September 30th, 2025'
```

## Locale Support

Format tokens like `MMMM`, `MMM`, `dddd`, `ddd`, `dd`, `Do`, `A`, and `a` are locale-aware and will display values based on the current locale setting.

```typescript
import { fdu, registerLocale } from "@pyyupsk/fdu";
import { es } from "@pyyupsk/fdu/locale/es";

registerLocale("es", es);

const date = fdu("2025-09-30");
date.locale("es").format("dddd, D [de] MMMM [de] YYYY");
// 'martes, 30 de septiembre de 2025'
```

See the [Localization Guide](/docs/localization) for more information about using different locales.
