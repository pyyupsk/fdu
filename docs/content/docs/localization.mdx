fdu provides comprehensive internationalization support with 11 built-in locales. Register and use locales globally or per-instance for localized date formatting.

## Supported Locales

fdu includes built-in support for 11 locales:

<Callout type="info" title="Under Development">
  Additional language support is currently under development and will be added in future releases.
</Callout>

- `en` - English (default)
- `es` - Spanish
- `fr` - French
- `de` - German
- `it` - Italian
- `pt` - Portuguese
- `ru` - Russian
- `ja` - Japanese
- `ko` - Korean
- `zh-cn` - Chinese (Simplified)
- `th` - Thai

## Using Locales

### Register a Locale

Import locale data from the source and register it:

```typescript
import { registerLocale } from "@pyyupsk/fdu";
import { es } from "@pyyupsk/fdu/locale/es";

registerLocale("es", es);
```

### Set Global Locale

```typescript
import { locale } from "@pyyupsk/fdu";

locale("es"); // Set Spanish as default
```

### Use Locale for Specific Instance

```typescript
import { fdu } from "@pyyupsk/fdu";

const date = fdu("2025-01-15");

date.locale("es").format("MMMM"); // 'enero'
date.locale("fr").format("MMMM"); // 'janvier'
date.locale("de").format("MMMM"); // 'Januar'
```

## Examples

### Spanish

```typescript
import { registerLocale, fdu } from "@pyyupsk/fdu";
import { es } from "@pyyupsk/fdu/locale/es";

registerLocale("es", es);

const date = fdu("2025-01-15").locale("es");

date.format("MMMM"); // 'enero'
date.format("dddd"); // 'miércoles'
date.format("D [de] MMMM [de] YYYY"); // '15 de enero de 2025'
```

### French

```typescript
import { registerLocale, fdu } from "@pyyupsk/fdu";
import { fr } from "@pyyupsk/fdu/locale/fr";

registerLocale("fr", fr);

const date = fdu("2025-01-15").locale("fr");

date.format("MMMM"); // 'janvier'
date.format("dddd"); // 'mercredi'
date.format("D MMMM YYYY"); // '15 janvier 2025'
```

### Japanese

```typescript
import { registerLocale, fdu } from "@pyyupsk/fdu";
import { ja } from "@pyyupsk/fdu/locale/ja";

registerLocale("ja", ja);

const date = fdu("2025-01-15").locale("ja");

date.format("YYYY年M月D日"); // '2025年1月15日'
date.format("dddd"); // '水曜日'
```

### Chinese (Simplified)

```typescript
import { registerLocale, fdu } from "@pyyupsk/fdu";
import { zhCn } from "@pyyupsk/fdu/locale/zh-cn";

registerLocale("zh-cn", zhCn);

const date = fdu("2025-01-15").locale("zh-cn");

date.format("YYYY年M月D日"); // '2025年1月15日'
date.format("dddd"); // '星期三'
```

## Custom Locale

You can create custom locale configurations:

```typescript
import { registerLocale, type LocaleConfig } from "@pyyupsk/fdu";

const customLocale: LocaleConfig = {
  name: "custom",
  months: [
    "January", "February", "March", "April", "May", "June",
    "July", "August", "September", "October", "November", "December",
  ] as const,
  monthsShort: [
    "Jan", "Feb", "Mar", "Apr", "May", "Jun",
    "Jul", "Aug", "Sep", "Oct", "Nov", "Dec",
  ] as const,
  weekdays: [
    "Sunday", "Monday", "Tuesday", "Wednesday",
    "Thursday", "Friday", "Saturday",
  ] as const,
  weekdaysShort: ["Sun", "Mon", "Tue", "Wed", "Thu", "Fri", "Sat"] as const,
  weekdaysMin: ["Su", "Mo", "Tu", "We", "Th", "Fr", "Sa"] as const,
  weekStart: 0, // 0 = Sunday, 1 = Monday, etc.
  ordinal: (n: number) => {
    const s = ["th", "st", "nd", "rd"];
    const v = n % 100;
    return n + (s[(v - 20) % 10] || s[v] || s[0]);
  },
  meridiem: (hour: number, minute: number, isLower: boolean) => {
    const m = hour < 12 ? "AM" : "PM";
    return isLower ? m.toLowerCase() : m;
  },
};

registerLocale("custom", customLocale);
```

## Locale Configuration

A locale configuration includes:

- `name` - Locale identifier
- `months` - 12 full month names
- `monthsShort` - 12 abbreviated month names (optional)
- `weekdays` - 7 full weekday names (Sunday first)
- `weekdaysShort` - 7 abbreviated weekday names (optional)
- `weekdaysMin` - 7 minimal weekday names (optional)
- `weekStart` - First day of week: 0-6 (optional, default: 0 = Sunday)
- `ordinal` - Function to format ordinal numbers (optional)
- `meridiem` - Function to format AM/PM (optional)

For complete API documentation, see [registerLocale()](/docs/api-locale/register-locale) and [locale()](/docs/api-locale/locale).
